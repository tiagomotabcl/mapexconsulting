Step-by-step instructions to get your website live at mapexconsulting.co.uk using GitHub Pages + Names.co.uk DNS. Takes about 20 minutes.

A GitHub — Create Repository & Upload Files

Step 1

Go to github.com and log in to your account (tiagomotabcl)

Step 2

Click the + icon (top right) → New repository

Repository name: mapexconsulting

Set to Public

Tick Add a README file

Click Create repository

Step 3 — Upload all 6 files

In your new repository, click Add file → Upload files

Drag all 6 files into the upload area:

index.html services.html consulting.html about.html contact.html CNAME

Scroll down and click Commit changes

⚠️ Important: Make sure the file called CNAME has NO file extension. It must be exactly CNAME, not CNAME.txt. If your computer added .txt, rename it before uploading.

Step 4 — Enable GitHub Pages

Go to your repository → Settings (tab at the top)

In the left sidebar, click Pages

Under "Build and deployment":

• Source: Deploy from a branch

• Branch: main · Folder: / (root)

Click Save

Step 5 — Set Custom Domain

Still on the Pages settings page, scroll down to "Custom domain"

Type: mapexconsulting.co.uk

Click Save

GitHub will show a yellow "DNS check in progress" — this is normal. It resolves after you set up DNS in the next section.

Step 6 — Enforce HTTPS

After the DNS check passes (may take 15-60 minutes), come back to this page and tick:

☑ Enforce HTTPS

This gives you a free SSL certificate. Your site will be served securely.

B Names.co.uk — Configure DNS Records

Step 1

Go to domains.names.co.uk and log in

Click on mapexconsulting.co.uk to manage it

Step 2

Find DNS Settings, DNS Management, or Advanced DNS

The exact menu name varies — look for "DNS Records" or "Manage DNS"

Step 3 — Delete default records

Delete any existing A records and CNAME records that came pre-configured. You're replacing them with GitHub's.

Step 4 — Add these DNS records

Add these 4 A records (these are GitHub Pages' IP addresses):

Type: A     Host: @     Value: 185.199.108.153     TTL: 3600
Type: A     Host: @     Value: 185.199.109.153     TTL: 3600
Type: A     Host: @     Value: 185.199.110.153     TTL: 3600
Type: A     Host: @     Value: 185.199.111.153     TTL: 3600

Then add this CNAME record for www:

Type: CNAME     Host: www     Value: tiagomotabcl.github.io     TTL: 3600

⚠️ Note on the CNAME value: Use your GitHub username: tiagomotabcl.github.io. If you created a new GitHub account called mapexconsulting, use mapexconsulting.github.io instead.

Step 5 — Save

Click Save or Update. Some providers save automatically per record.

C Wait & Verify

DNS Propagation

DNS changes take 5 minutes to 48 hours to propagate worldwide. Usually with Names.co.uk it's 15–30 minutes.

Test your site

After 15–30 minutes, open a browser and go to:

https://mapexconsulting.co.uk

If you see your homepage — congratulations, you're live! 🎉

Also test:

• https://www.mapexconsulting.co.uk (should redirect)

• https://mapexconsulting.co.uk/services.html

• https://mapexconsulting.co.uk/consulting.html

• https://mapexconsulting.co.uk/about.html

• https://mapexconsulting.co.uk/contact.html

If it doesn't work after 1 hour

1. Go back to GitHub → Settings → Pages — check if "DNS check" shows a green tick ✓

2. If not, double-check the 4 A records and CNAME in Names.co.uk — typos are the #1 cause

3. Try clearing your browser cache or using incognito mode

4. Check https://www.whatsmydns.net/#A/mapexconsulting.co.uk to see if DNS has propagated

✅ When everything works, you'll have:

• mapexconsulting.co.uk — your professional domain

• Free HTTPS (SSL) certificate from GitHub / Let's Encrypt

• Free hosting on GitHub Pages

• 5-page professional website with WhatsApp integration

• Total ongoing cost: ~£8/year (domain only)

D How to Update Your Site in the Future

To edit a page

1. Go to github.com/tiagomotabcl/mapexconsulting

2. Click the file you want to edit (e.g. index.html)

3. Click the pencil icon ✏️ (top right of the file)

4. Make your changes

5. Click Commit changes

6. Wait 1–2 minutes — GitHub rebuilds your site automatically

To replace a page completely

1. Click the file → pencil icon → Select All (Ctrl+A) → Delete

2. Paste the new content

3. Click Commit changes
